METASIM PLATFORM

A web-powered multi-engine, multi-server simulation API for distributed computation and easy access to WebGL



BUILD
LAUNCH
SIMULATE!
The MetaSim Platform intends to provide an open source cloud service & P2P driven solution for segmenting simulation components
 
This will allow for massively simulated parallel universe instancing! It will connect rendering clients or servers to our network of modular systems.
MetaSim will allow for deeply complex environments and cutting edge artificial intelligence to be integrated into your productions and upgraded as engine modules improve!

Holistic Components

Every module will be designed to work with other modules of every type and category. Pieces will talk to the renderer and to each other. Our goal is ultimate cross compatibility across type and version!

Modeling Nature

Representing the natural world is core to the design of environment modules, like plate-tectonic terrain systems, bioregoins and real climate models. Teams could use our system to model ecologically sustainable markets.

Real Science

We aim to facilitate interactions between brilliant developers and top-notch scientists around the globe! Cross disciplinary efforts would include everything from n-body galaxy collisions, mechanical modeling, to artificial life, agent systems and neural nets!

Make Learning Fun!

Our platform has obviously direct implications for producing games, but of a caliber which is not merely superficial. Our modeling engines will allow realistic experimentation at any scale. Learn by discovery!

Our Core Technologies
 
 

How do you build a nested object heirarchy and representative state model that allows for user viewing across an extremely large time and distance scale? Then allows for instancing by every potential end user?

We're still working out the engineering, but we're already on to the solutions. Every module is built around our central API, but must be independently designed to handle temporal constraints. This is how we've pieced our platform together:

 

 
 

Language
Agnosticism

Primary support is currently JavaScript, but Python, Java, C#, and Clojure module examples are in the works.

Cloud & VM
Deployment

Examples can be directly pushed to Heroku and run in unison to support the primary client and rendering server.

HTML5 with
WebGL

We want to integrate our awesome API with responsive modern design and as many WebGL engines as we can: ThreeJS, BabylonJS, Goo Engine, and experimental WebCL.

Github
Repositories

Being an open source project under MIT license means that the project is only as strong as the people supporting it, so join us!

Real-Time
Messaging

Designed for except handling and update resolves. WebSockets reduce latency by an iteration sync allowing concurrent engine updates.

 
We're looking for Partners!
 
 

Do you have what it takes to contribute to the MetaSim project? We are looking for volunteers, collaborators, venture backers, business partners, and compatible platforms to join in our efforts. The efforts will be challenging, but the rewards will be mind-boggling! Imagine empowering the crowd to generate new content and cross genre simulation experiences. That is our goal, help us get there!

Working together with our partners, we are hoping to come up with a hybrid approach between a federated object model and a blackboard agent system. Integrating a high level architecture into a modern browser API will solidify an engine standard which will allow deeper participation with scientific and GIS grade simulations. If successful, we hope to launch the next generation of the Multiverse MMO platform, and have it accessible directly through your browser using WebGL!

Please, if you are interested in helping, discussing, collaborating, or funding us, reach out and contact us!
